# MeditationApp — техническое задание

## 1. Назначение проекта

MeditationApp — мобильное Android-приложение для коротких дыхательных практик, расслабления, концентрации, восстановления и подготовки ко сну.

Главная идея приложения:

Открыл приложение → выбрал сессию → выбрал звуковой режим → нажал на дыхательный круг → следуешь дыханию и звуку → проходишь сессию до конца.

Приложение должно быть максимально простым, спокойным и понятным. Главный экран должен сразу быть рабочим инструментом, а не рекламной страницей.

## 2. Цель MVP

Создать первую рабочую Android-версию, которая позволяет пользователю:

- запускать дыхательную практику;
- видеть анимированный дыхательный круг;
- выбирать длительность сессии;
- выбирать звуковой режим;
- слышать генерируемый медитативный тон;
- использовать приложение без интернета;
- собрать и установить APK на Android-устройство.

## 3. Целевая аудитория

Основная аудитория:

- люди со стрессом;
- офисные сотрудники;
- программисты;
- родители;
- студенты;
- люди, которым сложно расслабиться;
- пользователи, которым нужны короткие практики на 5-20 минут.

Дополнительная аудитория:

- люди перед сном;
- водители после рабочего дня;
- пользователи, которым нужно спокойное приложение без лишних функций.

## 4. Основной пользовательский сценарий

1. Пользователь открывает приложение.
2. Видит главный экран MeditationApp.
3. Выбирает сессию.
4. Выбирает звуковой режим в прокручиваемом picker-окне.
5. Нажимает на дыхательный круг.
6. Запускается таймер сессии.
7. Круг увеличивается на вдохе, удерживается на задержке и уменьшается на выдохе.
8. Одновременно играет генерируемый звук.
9. Частоты, тональность и тембр меняются синхронно с фазами дыхания.
10. Повторный тап по кругу останавливает и сбрасывает сессию.
11. Паузы нет: смысл практики в том, чтобы пройти выбранное время полностью.

## 5. Функции MVP

В версию 1.0 входят:

- один главный экран;
- список медитационных сессий;
- анимированный дыхательный круг;
- старт/стоп прямо внутри круга;
- таймер сессии;
- текстовая подсказка текущей фазы дыхания;
- прокручиваемый sound picker в стиле старого Apple picker;
- генерация звука через Android AudioTrack;
- несколько популярных частот в каждом звуковом режиме;
- изменение звука при смене дыхательной фазы;
- оффлайн-работа;
- сборка debug APK.

Не входят в MVP:

- регистрация;
- аккаунты пользователей;
- облачная синхронизация;
- подписка;
- платежи;
- онлайн-библиотека;
- AI-генерация медитаций внутри приложения;
- MP3-файлы как основной источник звука;
- кнопка Pause.

## 6. Сессии MVP

В текущем MVP есть 3 базовые сессии:

1. Morning clarity — 5 минут
2. Deep focus — 12 минут
3. Sleep wind-down — 20 минут

Каждая сессия должна иметь:

- название;
- длительность;
- визуальное состояние выбранной сессии;
- связь с таймером;
- сброс таймера при смене сессии.

В следующих версиях можно добавить:

- Calm breath — 1 минута;
- Grounding reset — 3 минуты;
- Anti-stress after work — 5 минут;
- Child sleep story — 7-10 минут.

## 7. Звуковые режимы MVP

Звук генерируется в коде через Android AudioTrack. MP3-файлы в MVP не используются.

Текущие режимы:

- Focus;
- Relax;
- Reset;
- Harmony;
- Grounding;
- Clarity;
- Sleep.

Каждый режим должен содержать несколько популярных частот, используемых для разных дыхательных фаз.

Пример частот:

- Focus: 417 / 528 / 741 / 963 Hz;
- Relax: 174 / 285 / 432 Hz;
- Reset: 396 / 417 / 528 Hz;
- Harmony: 432 / 528 / 639 / 852 Hz;
- Grounding: 174 / 285 / 396 Hz;
- Clarity: 528 / 741 / 852 / 963 Hz;
- Sleep: 174 / 432 / 528 Hz.

Важно: частоты используются как художественный медитативный инструмент. Приложение не должно обещать медицинский эффект.

## 8. Логика дыхательной сессии

Базовый цикл:

- вдох — 4 секунды;
- задержка — 3 секунды;
- выдох — 5 секунд.

Итого один цикл: 12 секунд.

Анимация:

- во время вдоха круг плавно увеличивается;
- во время задержки остаётся большим или немного уменьшается;
- во время выдоха плавно уменьшается;
- текст фазы меняется синхронно с анимацией;
- звук меняет набор частот, тембр, пульсацию и гармоники синхронно с фазой.

Фазы:

- Inhale;
- Hold;
- Exhale.

Фаза Pause в MVP не используется.

## 10. Требования к дизайну

Стиль:

- ранний glossy Apple-style;
- светлые сиренево-фиолетовые тона;
- мягкие градиенты;
- круг как главный элемент управления;
- крупный центральный элемент;
- минимум текста;
- спокойные, мягкие карточки;
- без перегруженных плиток для звука.

Цветовая идея:

- почти белый фон;
- светлый lavender;
- мягкий violet;
- глубокий purple для заголовков;
- лёгкие стеклянные и glossy-эффекты.

Важно:

- верхнюю строку с дублирующей информацией о текущей сессии не показывать;
- sound picker должен быть не плитками, а прокручиваемым окном выбора;
- старт/стоп должен быть прямо в дыхательном круге;
- кнопка Pause не нужна.

## 11. Оффлайн-режим

Приложение должно работать без интернета.

В MVP внутри APK находятся:

- код генерации звука;
- список сессий;
- список звуковых режимов;
- UI и анимации.

Интернет не требуется для запуска и прохождения сессии.

## 12. Локальное хранение данных

В текущем MVP локальное хранение не обязательно.

В следующей версии можно добавить DataStore и хранить:

- последнюю выбранную сессию;
- последний выбранный звуковой режим;
- количество завершённых сессий;
- дату последней сессии;
- настройки звука.

Room Database в MVP не нужна.

## 16. Авторские права и безопасность

Так как MVP генерирует звук кодом, MP3-лицензии не требуются.

Если в будущих версиях появятся голос, музыка или сказки, запрещено использовать:

- чужую музыку;
- чужие медитации;
- фрагменты YouTube;
- аудиокниги;
- copyrighted ambient tracks;
- чужие сказки;
- узнаваемых персонажей без прав.

Разрешено использовать:

- оригинальные тексты;
- собственные записи;
- AI-generated voice при наличии коммерческих прав;
- AI-generated background music при наличии коммерческих прав;
- royalty-free звуки с лицензией для коммерческого использования.

Для каждого внешнего аудио в будущем нужно хранить:

- текст сценария;
- дату генерации;
- используемый сервис;
- название голоса;
- настройки генерации;
- подтверждение лицензии;
- исходный файл.

## 19. Интеграция с Project Alpha

MeditationApp может стать модулем Project Alpha.

Возможности:

- Alpha анализирует настроение пользователя;
- предлагает подходящую сессию;
- сохраняет историю практик;
- генерирует персональные медитации;
- создаёт сказки для ребёнка;
- хранит локальную память пользователя.

Пример будущего сценария:

1. Пользователь говорит Alpha: "Я устал после работы".
2. Alpha предлагает 5-минутную антистресс-сессию.
3. MeditationApp запускает нужную практику.
4. После завершения Alpha сохраняет результат.
5. Через месяц Alpha анализирует, какие практики лучше помогают.

## 20. Критерии готовности MVP

MVP считается готовым, если:

- приложение устанавливается на Android;
- открывается главный экран;
- пользователь может выбрать сессию;
- пользователь может выбрать звуковой режим;
- старт/стоп работает через дыхательный круг;
- паузы нет;
- работает анимация круга;
- таймер идёт до конца выбранной сессии;
- звук генерируется через AudioTrack;
- частоты меняются при смене дыхательной фазы;
- приложение работает без интернета;
- APK собирается командой `./gradlew assembleDebug`;
- нет критических ошибок при запуске.

## 21. Главное ограничение MVP

Главное правило:

Не усложнять.

Первая версия должна быть простой, красивой и рабочей. Если новая функция мешает главному сценарию "выбрал сессию → нажал круг → прошёл практику", её нужно отложить.
